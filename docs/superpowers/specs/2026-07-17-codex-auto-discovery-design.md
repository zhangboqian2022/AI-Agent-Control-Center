# Codex 自动任务发现与桌面交付设计

日期：2026-07-17

## 目标

让 AACC 默认只展示 Codex，并自动列出本机正在运行或最近更新的 Codex 任务；用户可在设置中勾选需要展示的代理类型。面板保持在普通桌面窗口之上，并提供可直接安装的 DMG。

## 数据边界

发现器只读取 `~/.codex/session_index.jsonl` 和 `~/.codex/process_manager/chat_processes.json` 的任务 ID、标题、更新时间与进程 PID。不会读取会话 JSONL 的对话、提示词、代码或命令内容。PID 存活表示“运行中”；无存活 PID 的最近任务只显示“状态未知”，不伪造等待批准或完成状态。

## 行为

每两秒轮询一次本地索引。活动任务按更新时间优先，随后是最近任务，最多展示 20 张卡片。任务名称来自 Codex 任务标题；没有标题时使用任务 ID 短码。自动任务以 `codex:<conversation-id>` 作为稳定 ID，并写入现有 SQLite 状态库。

设置中保存代理筛选：Codex、Claude Code、Kimi Code、Z Code。初始值仅为 Codex；手动配置的代理任务仅在对应类型被勾选时显示。点击 Codex 自动任务只会唤起 Codex 应用，因 Codex 没有公开的精确任务聚焦接口。

## 桌面与安装

窗口继续使用 macOS 的置顶窗口标志；首次无位置记录时固定在主显示器右上角并保留用户拖动位置。构建脚本生成 `AACC-1.0.0.dmg` 到桌面，DMG 内含已签名的 `AACC.app`。

## 验证

为本地 JSONL/进程解析、动态任务注册、筛选、首次右上角定位及 DMG 构建脚本添加自动化测试。最终运行完整测试、lint、类型检查、构建和实际 DMG 挂载检查。
