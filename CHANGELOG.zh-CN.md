# 更新日志

## 1.3.2 — 2026-07-22

- [安全] 占位形态的 API token 改为按前缀拒识（`change-me`、`replace-`、`your-token`、`placeholder`），示例配置 token 置空——加载后必定轮换为全新随机凭证，不再可能以公开常量运行。
- [体验] 隐藏某个 Agent 品牌现在跨重启保留：新品牌可见性播种改为一次性迁移键，不再每次启动强制恢复 Kimi 品牌。
- [API] `/api/v1/reload-config` 返回 501 Not Implemented，不再用误导性的 200。
- [稳定性] `aacc doctor` 与 app 通过统一函数解析运行时数据库路径，诊断检查的就是 app 实际挂载的文件。
- [体验] 轮换 API token 不再自动写剪贴板：新凭证在弹窗中展示，点击"复制"才写入。
- [性能] 过期状态历史清理节流为每小时一次并为 `created_at` 加索引；分组与顺序未变时面板跳过每秒的卡片布局重建。
- [稳定性] 任务状态订阅者异常改为记录日志，不再静默吞掉。
- [文档] 修正 README 中"安装器会运行测试"的表述（默认跳过，`AACC_RUN_TESTS=1` 开启），补齐 1.3.1 测试报告。

- [体验] 卡片右键菜单移除语音输入与按键注入项；相关自动化能力仍可通过全局热键使用。

## 1.3.0-rc.6 — 2026-07-21

## 1.3.0-rc.5 — 2026-07-21

- [稳定性] 修复 Kimi 会话在回合内长时间静默（模型响应慢、长工具调用或上下文汇总）时被误判为空闲：回合仍在进行时，在有界的活跃回合窗口内（默认 30 分钟）保持“运行中”，不再因 90 秒无文件活动而掉成空闲；超过窗口仍会回落为空闲，避免崩溃会话永远显示运行中。
- [体验] 新增“关于 AACC”对话框（面板头部 ⓘ 按钮），显示当前运行版本与对应的 DMG 安装包名称。

## 1.3.0-rc.2 — 2026-07-20

- [体验] 将任务卡改为紧凑横向布局：左侧保留醒目大灯，工具名显示为小徽标，任务名称突出，总计时上移，并显示一行实时短概括。
- [体验] 面板会随可见任务自动拉长或收短，最高限制为当前屏幕可用高度的 80%，超过后才启用内部滚动。
- [稳定性] Codex 自动发现改为每 5 秒查询一次，并用固定短语概括分析、改代码、测试、构建、检查、查询、执行命令与完成状态。
- [稳定性] 多个短回合与等待状态共用一次完整运行计时；终态冻结总用时，只有终态任务重新开始时才归零。
- [稳定性] 识别当前 Codex 的输入、命令、补丁与权限审批事件；通过带缓存的增量元数据扫描恢复活动尾窗外的冷启动计时；等待任务与其他活跃任务同等优先。
- [隐私] 状态概括不会把提示词、回答、命令、凭证或文件内容复制到任务消息、日志或面板。
- [稳定性] 窗口退出前主动停止 GUI 刷新定时器，并让任务管理器支持幂等关闭，避免已排队的 Qt 刷新访问已关闭的 SQLite 连接。

## 1.3.0-rc.1 — 2026-07-20

- [Security] 配置改为原子写入，自动修复无效 API Token，并拒绝带空白字符的凭证；强制配置与数据库私有权限，新增本地凭证轮换，并加强日志脱敏与 AppleScript 文本隔离。
- [Stability] 完整桌面自动化事务进入有界单工作线程，超时后的排队或待注入动作会在延迟输入前取消，Qt 不再阻塞；任务起始时间保持稳定，重复历史被抑制，SQLite 历史有上限，Codex 发现故障可见且可恢复。
- [Stability] 新增 PID 身份校验、单实例锁、包装器协作式子进程清理、辅助功能权限引导、事件 tap 恢复及 Adapter 断开唤醒。
- [Breaking] 源码安装器不再把 CLI 链接到仓库 `.venv`，而是安装到 Application Support 下的运行时专用环境。
- [Delivery] QSS 作为资源打包，生产依赖严格来自锁文件，加入当前 Codex 格式脱敏夹具；DMG 支持复用构建，并加入成对的 Developer ID/公证配置校验及明确的 ad-hoc 预发布标识。
